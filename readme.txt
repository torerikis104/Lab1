se og til en viss grad manipulere innholdet på en webiside, dette er View. View i min oppgave er altså resultatet av min Model, brukers 
Kontroler og koblingen mot internet og en webserver.

1)	Classes kan brukes flere ganger i et dokument, mens id er unikt. 
Det er en fordel å bruke class når man skal bruke et element ofte i dokumentet, feks 
om du vil gi alle <p> samme styling.
id er en unik beskrivelse av ett element, som bare kan brukes en gang i dokumentet. Vil jeg at alle
<p> skal være centrert, kan jeg bruke class, vil jeg bare ha en av dem sentrert, kan jeg gi den <id>.

2)	I deloppgave 0 blir listen veldig lang, og man må scrolle ned for å finne språket man leter etter.
Dette kan gjøre at en side laster inn saktere, og det gjør hele siden mer uoversiktlig og rotete.
Dersom man bruker <form> blir det hele mer oversiktlig og man kan feks "søke"
etter riktig språk ved å aktivere boksen i rullegardinet og trykke P for å få rulle ned til alle språk som 
begynner med P. Bakdelen med dette kan være når man er en bruker som ikke forstår hvordan man gjør dette.

3) Wikipedia bruker: <table>
						<td>
						</td>
					 </table>
<td> tagen er en standard celle i et html table.
Slik får de alle språk som begynner på A, "på samme sted" på siden, slik at det blir enklere for bruker å se dem.
Mer oversiktlig og forståelig.

4)	<div> tagen er en block-element tag, og kan brukes som en container for å gruppere andre HTML elementer. 
Brukt sammen med CSS, kan <div> brukes til å style "block-level" elementer med innhold.
	<span> tagen er et inline-element, som kan brukes som en container for tekst. Brukt sammen med CSS
kan man style deler av en tekst. eks: <p>Dette er <span>blått</span></p> - her kan man ved å bruke CSS
få teksten "blått" til å faktisk ha blå farge i det som blir lesbart i browser eller applikajsonen.
